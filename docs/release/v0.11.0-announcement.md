## GitHub Release body

```markdown
## Tyra v0.11.0 — AI self-correction

The compiler now catches what it used to silently miss, reports it
machine-readably, and programs report their own failures cleanly.

### What's new

**AI self-correction (ADR-0028/0029)**
- Imported module calls are fully type-checked (was silently untyped).
  `String + string.from_byte(x)` now fires E0305 instead of crashing codegen.
- New diagnostics: E0318 (unknown module function, catches typos),
  E0319 (non-displayable argument to print family).
- `fn main() -> Result<Unit, E>` returning `Err` reports `error: …` on
  stderr and exits 1 (previously silently exited 0).

**Machine-readable diagnostics (ADR-0026)**
- `tyra check/build --error-format json` emits NDJSON on stderr.
- Three record types: `diagnostic`, `error`, `summary`.
- Stderr is guaranteed NDJSON-only on every code path, including usage
  errors and ICEs. Built for agent self-correction loops.

**stdlib additions (ADR-0027)**
- `string.chars` / `char_at` / `char_code` / `from_char_code` — USV character API.
- `list.sort` (Int) / `list.sort_str` (String) — stable ascending sort.
- `eprint` / `eprintln` fixed to write to stderr (previously wrote stdout).

**ai-gen benchmark**
- Multi-seed sweep with v0.11.0: **tyra+spec 88.7% mean** (3 seeds × 100 prompts = 300 runs).
  any_pass 98%, all_pass 77%. No same-condition baseline exists for run56; the previously
  published 77% used a different (stale v0.10) binary with single-seed only.

### Breaking changes

- `string.to_upper` / `to_lower` → `to_ascii_upper` / `to_ascii_lower`
- `Err`-returning main now exits 1 instead of 0
- Module calls that previously compiled silently may now fail with real type errors

### Install / upgrade

```bash
# curl|sh
curl -fsSL https://raw.githubusercontent.com/tyra-lang/tyra/main/scripts/install.sh | sh

# Homebrew
brew upgrade tyra-lang/tap/tyra

# Specific version
curl -fsSL https://raw.githubusercontent.com/tyra-lang/tyra/main/scripts/install.sh | sh -s -- --version v0.11.0
```

Full changelog: [CHANGELOG.md](https://github.com/tyra-lang/tyra/blob/main/CHANGELOG.md)
```

## Hacker News / lobste.rs post

**Title:** Tyra v0.11.0 – AI self-correction: machine-readable diagnostics, 88.7% LLM first-try pass rate (3-seed)

**Body:**

```
Tyra is a statically-typed language (Rust compiler, LLVM backend) designed
for human-AI collaboration. v0.11.0 focuses on making the compiler useful
as feedback signal in agent self-correction loops.

Three things shipped:

1. AI self-correction diagnostics — module calls are now fully type-checked
   (previously silently untyped, letting `String + string.from_byte(x)` slip
   past the checker and crash codegen). New E0318/E0319 cover unknown function
   names and non-displayable print arguments. Err-returning main reports to
   stderr + exits 1.

2. --error-format json — tyra check/build now emit NDJSON diagnostics on
   stderr. Three record types (diagnostic / error / summary), guaranteed
   NDJSON-only on every path including usage errors and ICEs. An agent loop
   can do: generate → tyra check --error-format json → parse stderr → fix.

3. ai-gen multi-seed sweep: 88.7% mean — we benchmark how often frontier
   models generate Tyra code that compiles and runs correctly on the first
   try (3 seeds × 100 prompts = 300 runs, Claude Sonnet 4.6 with spec
   injected). After v0.11 fixes: 88.7% mean, up from 77% with the stale
   v0.10 binary. any_pass (≥1 seed) = 98%. For comparison: Ruby 99%,
   Crystal 96%, Go 81% (seed 1 only — not yet multi-seed). Tyra's multi-seed mean
   exceeds Go's seed-1 point estimate; a same-condition comparison is still pending.

Also: USV character API (chars/char_at/char_code/from_char_code), list.sort /
sort_str, to_ascii_upper/lower rename (breaking), eprint stderr fix, spec 0.11.

GitHub: https://github.com/tyra-lang/tyra
Install: curl -fsSL https://raw.githubusercontent.com/tyra-lang/tyra/main/scripts/install.sh | sh
Docs: https://github.com/tyra-lang/tyra/blob/main/llms.txt (LLM-readable spec)
